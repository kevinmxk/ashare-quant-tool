# 局域网传输说明

当前会话环境无法直接把代码发到 `192.168.137.102`，但已经准备好了本机可执行脚本：

实际脚本位置：

- [deploy_to_lan.ps1](C:/Users/Kevin/Documents/Codex/2026-05-19/a-a/scripts/deploy_to_lan.ps1)

## 方式 1：SMB 共享目录

如果 `192.168.137.102` 开了共享目录，例如共享名是 `projects`，可以在项目根目录运行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy_to_lan.ps1 -Mode smb -RemoteShare projects -RemotePath ashare-quant-tool
```

如果共享需要账号密码：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy_to_lan.ps1 -Mode smb -RemoteShare projects -RemotePath ashare-quant-tool -Username your_user -Password your_password
```

## 方式 2：SCP / SSH

如果目标主机开了 SSH：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\deploy_to_lan.ps1 -Mode scp -Username your_user -SshRemotePath /opt/ashare-quant-tool
```

## 脚本行为

- 会同步整个项目目录
- 会自动排除 `.git`、`__pycache__`、`.venv`、`dist`
- `smb` 模式使用 `robocopy /MIR`
- `scp` 模式使用递归复制

## 建议

第一次建议先用一个临时目录，例如：

- `ashare-quant-tool-test`

确认目标机上权限和目录都没问题，再切正式目录。
