# AKShare 接入与兼容性说明

## 当前策略

系统默认使用 `ASHARE_QUANT_PROVIDER=auto`：

- 当前机器安装了 `akshare` 且接口可调用时，优先使用真实数据
- 当 `akshare` 未安装、接口异常或上游暂时失败时，自动回退到 `Mock` 数据源

这样做的目的是让项目迁移到新电脑时更平滑：

- 有依赖时直接跑真实数据
- 没有依赖时至少能先把系统启动起来

## 已做的兼容性处理

### 1. 股票代码标准化

系统统一接受以下格式：

- `000001`
- `sz000001`
- `SZ000001`
- `sh600519`
- `bj430047`

进入分析层之前，都会被清洗成统一的 6 位或北交所代码格式。

### 2. 列名别名清洗

AKShare 不同版本、不同接口文档和上游源之间，常见差异包括：

- `5 分钟涨跌` 和 `5分钟涨跌`
- `代码` 和 `股票代码`
- `名称` 和 `股票简称`
- `金额` 和 `成交额`

当前系统已经把这类字段差异统一收敛在 `akshare_cleaner.py` 中。

### 3. 历史行情参数兼容

`stock_zh_a_hist` 使用：

- `symbol=000001`
- `period=daily`
- `adjust=qfq`
- 自动计算 `start_date`
- 自动设置 `end_date`

这样可以减少不同机器、不同日期边界下的行为差异。

### 4. 轻量缓存

实时快照接口增加了短 TTL 内存缓存，减少：

- 重复请求
- 网络抖动影响
- 上游限流风险

## 迁移到新电脑时建议

1. 安装 Python `3.9` 或更高版本
2. 优先创建独立虚拟环境
3. 执行 `pip install -e .`
4. 复制 `.env.example` 为 `.env`
5. 首次启动时先跑 `python scripts/akshare_parser_check.py`
6. 若需要 API，再安装并启动 `FastAPI` 相关依赖

## 已知边界

- `stock_zh_a_spot_em` 不直接提供稳定的“所属行业”字段，当前字段为空时属于正常情况
- `stock_zh_a_hist` 是日线数据，盘中不应把它当作实时 K 线
- 免费源仍可能因为上游网页变动而失效，因此需要后续补 `Tushare` 和 `BaoStock` 作为冗余源
