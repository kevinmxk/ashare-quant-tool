# 本地持久化缓存层设计说明

## 当前方案

`V1` 先采用 `SQLite` 做本地持久化缓存，而不是直接强依赖 `DuckDB/Parquet`。

这样做的原因是：

- Python 标准库自带 `sqlite3`
- 跨电脑迁移时几乎零额外依赖
- 对当前“行情快照 + 日线缓存”场景已经足够

后续如果你需要更强的离线研究和批量回测能力，可以再加：

- `DuckDB` 作为分析型查询层
- `Parquet` 作为归档层

## 缓存对象

当前缓存三类数据：

1. `quote_cache`
2. `daily_bar_cache`
3. `universe_batches`

### quote_cache

存最新股票快照：

- 最新价
- 涨跌幅
- 换手率
- 成交额
- 量比
- 估值字段

### daily_bar_cache

存股票历史日线：

- 开高低收
- 成交量
- 成交额

### universe_batches

存一次股票池抓取的顺序和时间，解决“只缓存单股快照但丢失榜单顺序”的问题。

## 当前接入方式

增加了 `PersistentCacheMarketDataProvider`，它包裹真实 provider：

- 先读缓存
- 缓存过期时再请求真实源
- 请求成功后回写缓存
- 真实源失败时，允许回退到旧缓存

## 配置项

`.env` 中可用的缓存配置：

- `ASHARE_QUANT_PERSISTENT_CACHE_ENABLED`
- `ASHARE_QUANT_PERSISTENT_CACHE_PATH`
- `ASHARE_QUANT_PERSISTENT_QUOTE_TTL_SECONDS`
- `ASHARE_QUANT_PERSISTENT_BAR_TTL_SECONDS`
- `ASHARE_QUANT_PERSISTENT_CACHE_ALLOW_STALE_ON_ERROR`

## 当前默认值

- 快照 TTL：`900` 秒
- 日线 TTL：`43200` 秒
- 默认缓存文件：`data/cache/market_cache.sqlite3`

## 迁移到新电脑时的建议

1. 默认保留持久化缓存开启
2. 首次部署后先运行 `python scripts/cache_smoke_test.py`
3. 如果你想把缓存文件放到别的盘，改 `ASHARE_QUANT_PERSISTENT_CACHE_PATH`
4. 如果你担心盘中数据太旧，可以缩短 `QUOTE_TTL`
5. 如果你更在意抗故障能力，可以保留 `ALLOW_STALE_ON_ERROR=true`
