# A 股量化分析工具 V1 技术设计书（上半部分）

## 1. 项目目标

本项目的目标不是做“自动赚钱机器”，而是做一个对个人投资者真正有用的量化辅助系统。它需要满足以下几点：

- 能基于免费或低成本数据源完成 A 股日常研究
- 能形成可解释的选股评分，而不是黑盒预测
- 能辅助判断买点、卖点和持仓观察优先级
- 能逐步演进到回测、预警、自选池监控等场景

V1 阶段的定位是：

- 面向个人研究场景
- 以日线和分钟级辅助分析为主
- 强调可扩展、可替换的数据接入层
- 优先构建研究闭环，而不是直接对接实盘交易

## 2. 产品边界

V1 要解决的问题：

- 构建股票基础池与自选池
- 生成一套可解释的量化评分
- 对单只股票输出趋势、强弱、流动性、风险的诊断
- 支持通过 API 方式被网页、桌面端或后续策略模块调用

V1 不解决的问题：

- 不做自动下单
- 不承诺全市场毫秒级实时刷新
- 不依赖单一免费网页源
- 不做复杂机器学习预测

## 3. 核心原则

### 3.1 多数据源容错

免费行情源存在以下共性风险：

- 接口字段会变化
- 频率过高时容易触发反爬
- 个别时段可能返回空值或异常值

因此系统必须使用“统一数据接口 + 多 Provider 回退”设计，而不是把策略逻辑直接绑定到单一站点。

### 3.2 因子与评分可解释

每一条评分结果都需要能回答：

- 为什么这只股票分数高
- 是趋势强，还是板块强，还是成交活跃
- 风险扣分来自哪里

### 3.3 研究优先于交易

V1 的先后顺序是：

1. 稳定取数
2. 清洗与标准化
3. 计算因子
4. 生成评分
5. 验证有效性
6. 再考虑盘中提醒与实盘联动

## 4. 总体架构

V1 采用四层架构：

### 4.1 数据接入层

职责：

- 对接 `AKShare`、`Tushare`、`BaoStock`
- 做字段标准化
- 对异常值和空值做基础校验
- 提供统一的行情查询接口

输入：

- 股票代码列表
- 查询日期区间
- 刷新模式

输出：

- 标准化后的股票快照
- 历史 K 线序列
- 市场概览数据

### 4.2 分析引擎层

职责：

- 计算基础技术因子
- 计算流动性与风险指标
- 生成评分与解释标签

V1 暂定因子：

- 动量因子
- 趋势因子
- 流动性因子
- 估值因子
- 波动风险因子

### 4.3 服务编排层

职责：

- 调用数据源和分析引擎
- 生成市场排名结果
- 对外提供统一接口

### 4.4 展示层

职责：

- 提供 API 或简单页面
- 展示股票榜单、单股诊断、自选池结果

V1 先以 API 为主，前端暂用简单页面或后续接入 `Streamlit`。

## 5. 数据源策略

### 5.1 AKShare

定位：

- 免费版主入口
- 适合行情快照、分钟线、板块与市场数据

优点：

- Python 生态友好
- 社区使用广
- 对接成本低

风险：

- 部分底层依赖第三方网页数据源
- 字段格式可能随上游变化
- 高频访问时稳定性一般

在 V1 中的角色：

- 主行情快照源
- 主分钟级观察源

### 5.2 Tushare

定位：

- 结构化日线与日度指标的高质量补充源

优点：

- 数据结构统一
- 适合做日频分析与因子研究

限制：

- 部分实时接口需要权限
- 免费额度与调用方式存在约束

在 V1 中的角色：

- 收盘后分析主源
- 日度估值、换手率等因子辅助源

### 5.3 BaoStock

定位：

- 历史数据补源

优点：

- 历史研究友好
- 使用成本低

限制：

- 更偏历史研究，不适合作为盘中主引擎

在 V1 中的角色：

- 历史补数
- 日线回测备用源

## 6. 模块拆分

### 6.1 `providers`

负责不同数据源接入，统一输出标准对象。

建议子模块：

- `base.py`
- `mock_provider.py`
- `akshare_provider.py`
- `tushare_provider.py`
- `baostock_provider.py`
- `factory.py`

### 6.2 `analysis`

负责因子计算和评分。

建议子模块：

- `factors.py`
- `scoring.py`
- `signals.py`

### 6.3 `services`

负责业务编排。

建议子模块：

- `market_service.py`
- `watchlist_service.py`

### 6.4 `api`

负责对外接口。

建议子模块：

- `main.py`
- `schemas.py`

## 7. V1 首批接口设计

### 7.1 市场股票池

用途：

- 返回可分析的股票列表

接口：

- `GET /universe`

### 7.2 排名榜单

用途：

- 返回当前评分靠前的股票

接口：

- `GET /rankings`

### 7.3 单股诊断

用途：

- 返回单只股票的评分结果、因子结果和解释

接口：

- `GET /stocks/{symbol}`

### 7.4 健康检查

接口：

- `GET /health`
